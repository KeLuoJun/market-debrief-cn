# market-debrief-cn · Skill 设计文档

> **定位**：A股收盘后的深度日报生成工具。核心目标不是「描述今天发生了什么」，而是「理解今天市场在定价什么、定价是否合理、明天可能发生什么」。

---

## 一、设计理念

### 核心哲学

与 huashu-data-pro 的「数据分析」不同，market-debrief-cn 的核心产品是**一份每日更新的、有独立判断的市场解读报告**，而非对用户上传数据的分析。因此，其设计哲学有三个关键差异：

- **数据自驱动**：不依赖用户上传数据，由 skill 自行通过 akshare 和 tavily 拉取当日市场数据与新闻
- **分析框架固定**：报告结构相对稳定，每期框架一致，便于读者建立阅读习惯与横向比较
- **深度优先**：宁可少写几个模块，也要把写到的每个模块做到有真实判断、有量化锚点、有因果逻辑

### 精简原则

原框架（九层）在实践中存在两个问题：盘中时序数据无法获取（akshare 不提供分钟级历史切片），且九个模块全部写深将导致单期报告体量过大、阅读疲劳。本设计将其精简为**六个分析模块**，通过以下方式实现减量不减深度：

- **删除**：Section 3「盘中时序解剖」（数据不可得）
- **合并**：Section 6「技术形态」+ Section 7「估值锚点」→ 合并为单一模块「技术与估值」
- **合并**：Section 8「次日预判」+ Section 9「历史镜像」→ 合并为单一模块「预判与验证」
- **取消封面**：原 Section 0 封面改为 HTML 顶部仪表盘横条，不占独立模块位置
- **保留精华**：各模块核心的量化方法论和判断逻辑完整保留

---

## 二、触发逻辑

用户输入以下类型的指令时触发本 skill：

- 明确指令：「生成今日A股日报」「做今天的市场复盘」「出一期盘脉日报」
- 简化指令：「A股复盘」「今日市场分析」「跑一期日报」
- 日期指定：「帮我做2025年XX月XX日的日报」（可指定历史交易日）

触发后，skill 自动判断目标日期（默认为最近一个已收盘的交易日），无需用户手动输入任何数据。

## 三、数据来源说明

### akshare（市场行情数据）

用于获取所有结构化的量化数据，包括但不限于：

- 主要指数收盘价、涨跌幅、成交额（沪指、深成、创业板、科创50）
- 申万一级行业涨跌幅与净流入额
- 全市场资金流向（超大单/大单/中单/小单净流入，按行业）
- 主要指数历史估值数据（PE TTM、PB）
- 涨停/跌停数量、封板率
- 两融余额、北向资金（历史数据）
- 主要指数历史均线数据（MA5/20/60/250）
- 龙虎榜数据（触发条件、席位、净买入）
- 10年期国债收益率

### tavily（新闻与事件数据）

用于获取驱动当日行情的非结构化信息，包括：

- 当日A股相关重要新闻（政策、产业、监管、宏观经济数据发布）
- 主力板块的驱动事件（如AI、新能源、半导体等主题的核心催化）
- 隔夜外盘动态（美股、港股、大宗商品）
- 重要宏观数据（CPI、PMI、MLF操作等，当日或近期发布）

## 四、多 Agent 架构设计

本 skill 采用「主协调 Agent + 三个并行分析 Subagent + 渲染 Agent」的四层架构，参照 huashu-data-pro 的多专家并行模式，但针对日报场景做了以下适配：

- **角色固定化**：不需要像 huashu-data-pro 那样动态选角，每期的三个 Subagent 角色固定，对应固定分析模块
- **数据预采集**：主 Agent 在启动 Subagent 之前完成所有数据采集和预处理，Subagent 只做分析，不做数据拉取

### 架构总览

```
┌─────────────────────────────────────────────┐
│          主协调 Agent (Orchestrator)          │
│  ① 确认目标日期                               │
│  ② 调用 akshare 采集全部行情数据              │
│  ③ 调用 tavily 采集新闻与事件                 │
│  ④ 结构化数据，写入共享上下文                  │
│  ⑤ 启动三个并行 Subagent                      │
│  ⑥ 收集结果，传入渲染 Agent                   │
└─────────────────────────────────────────────┘
          ↓ 并行启动（run_in_background=true）
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│  Subagent A  │  │  Subagent B  │  │  Subagent C  │
│  宏观与情绪  │  │  板块与资金  │  │ 技术·估值·预判│
└──────────────┘  └──────────────┘  └──────────────┘
          ↓ 全部完成后
┌─────────────────────────────────────────────┐
│           渲染 Agent (Renderer)               │
│  统一综合三个 Subagent 的分析结论              │
│  生成最终交互式 HTML 报告                     │
└─────────────────────────────────────────────┘
```

### 主协调 Agent（Orchestrator）

**职责**：数据采集与流程调度，不做任何分析判断。

执行步骤：
1. 解析用户意图，确认目标交易日期
2. 调用 akshare 拉取当日及近期历史行情数据（具体字段详见各模块说明）
3. 调用 tavily 搜索当日关键事件、新闻摘要
4. 将所有原始数据整理为结构化 JSON，包含各 Subagent 所需的数据子集
5. 以 Task 工具同时启动三个 Subagent（背景并行执行）
6. 等待全部 Subagent 完成，收集各自输出的 JSON 格式分析结论
7. 将三份分析结论 + 原始数据传入渲染 Agent

**输出物**：面向 Subagent 的结构化数据包（JSON），以及触发渲染 Agent 的完整上下文。

---

### Subagent A：宏观事件解读师 × 情绪量化分析师

**分析模块**：Module 1「宏观定价扫描」+ Module 2「市场情绪温度计」

**接收数据**：tavily 新闻摘要、北向资金、成交额历史序列、涨跌停数量、两融余额变化、换手率

**核心分析任务**：

针对 Module 1：
- 从 tavily 的新闻摘要中筛选出对盘面有真实传导影响的有效事件（过滤无关噪音）
- 对每个有效事件判断传导路径（事件→受影响资产类别→盘面印证）
- 对每个事件给出「定价完成度」判断：已充分定价 / 未充分定价 / 过度定价
- 对当日最关键的 1-2 个事件额外提炼“预期差”：市场原预期、今日新增信息、盘面反应、明日验证点、证伪条件
- 更新宏观三维坐标系：经济动能方向、流动性环境、风险偏好各打一个状态标签，并给出数据依据

针对 Module 2：
- 基于多维指标计算今日综合情绪分（0-100），各指标权重参见下文框架说明
- 分别估算「散户情绪代理分」和「机构情绪代理分」，判断两者的分歧或共识方向
- 与近20日均值对比，输出5日情绪趋势方向

**输出格式**：JSON，包含有效事件列表（含定价完成度）、宏观三维坐标、情绪综合分、散户/机构分项、5日趋势描述。

---

### Subagent B：板块策略师 × 资金流向分析师

**分析模块**：Module 3「板块结构性分析」+ Module 4「资金路线图」

**接收数据**：申万一级行业涨跌幅、行业净流入额、超大/大/中/小单分行业数据、涨停股及连板数、龙虎榜席位数据

**核心分析任务**：

针对 Module 3：
- 输出行业强弱全景（强势/次强/中性/弱势四分类）及今日成长/价值风格相对强弱
- 对今日涨幅最高的 TOP 2 板块做深度解剖：驱动力来源分类、轮动生命周期定位（启动/加速/高潮/分化/衰减）、资金主体识别（游资/机构/量化）
- 分析涨停板生态：总数、首板/连板分布、封板率、炸板数、主题集中度，给出当日赚钱效应的综合判断

针对 Module 4：
- 计算全市场超大单 vs 小单净流入方向，判断机构/散户行为分歧类型
- 绘制资金迁移路线（净流出板块 → 净流入板块），计算资金迁移的性质（防御切换/进攻切换/内部轮动）
- 解读龙虎榜重要席位：区分知名游资/机构/量化，判断主力意图

**输出格式**：JSON，包含行业热力分类、TOP 2 板块深度解剖结论、涨停生态快照、资金迁移路线图、龙虎榜解读。

---

### Subagent C：技术分析师 × 估值研究员 × 情景规划师

**分析模块**：Module 5「技术形态与估值」+ Module 6「次日预判与历史镜像」

**接收数据**：主要指数收盘价序列（近250日）、成交量序列、PE/PB 历史百分位、国债收益率

**核心分析任务**：

针对 Module 5（技术部分）：
- 计算主要指数当前与 MA5/20/60/250 的位置关系，判断均线排列结构
- 量价关系组合分析（放量上涨/缩量上涨/放量下跌/缩量下跌等），计算今日成交量与5日均量之比
- 识别今日 K 线形态，给出对应的概率性解读（非简单描述，要输出未来1-3日的方向概率估算）
- 计算关键支撑位与压力位，注明计算依据（均线/斐波那契/成交密集区）
- 计算日内涨幅（收盘/开盘-1）与隔夜涨幅（开盘/前收-1），判断资金行为性质

针对 Module 5（估值部分）：
- 输出沪深300、创业板指、科创50、中证500当前 PE(TTM) 及其历史百分位
- 计算股债收益差（ERP = 沪深300股息率 - 10年期国债收益率），与历史均值对比，判断股票相对债券的性价比

针对 Module 6：
- 基于上述技术、情绪、估值综合输入，给出次日三情景预判（核心/乐观/悲观），概率之和为100%，每个情景必须附带：可观测的触发条件（开盘30分钟内可验证）、预期指数区间、简要操作参考
- 每个情景必须说明它在验证哪条市场主叙事，以及什么信号会让该情景迅速失效
- 寻找当前市场状态（估值百分位 + 情绪分 + 趋势状态）的历史镜像期，说明相似点与关键差异
- 对上一期预判做准确度回顾（若有历史记录），输出命中情景及根因分析

**输出格式**：JSON，包含技术状态矩阵、K 线形态解读、关键位计算、量价组合判断、估值水位表、ERP 分析、三情景预判表、历史镜像描述、上期预判回顾。

---

### 渲染 Agent（Renderer）

**职责**：将三个 Subagent 的分析结论整合为一份单文件交互式 HTML 日报。

**设计原则**：参照 huashu-data-pro 的「管理型分析师整合视角」——最终报告不以 Agent 视角呈现，而是作为一份统一的研究报告，按模块顺序组织内容。

渲染 Agent 的核心工作包括：
- 检查三份 JSON 是否存在数据冲突或逻辑矛盾，若有则以 Subagent C（技术/估值）的数据为准
- 将分析结论转化为 HTML 模块，每个模块的标题使用**结论式句式**（如「资金从消费向科技大迁移，赚钱效应持续修复」），而非描述式（如「板块资金分析」）
- 生成文本叙事 + 数据可视化的混排布局

---

## 五、HTML 报告设计规范

### 整体风格

固定采用「Financial Times 风格变体」——三文鱼粉底色系的温暖权威感，适配金融叙事类报告，让每期报告具有稳定的品牌识别度（区别于 huashu-data-pro 每次随机风格）。

核心色彩系统：
- 页面底色：温暖米白（#FAF7F2）
- 强调色：深砖红（#C0392B），用于上涨/利多标注
- 下行色：暗蓝灰（#2C3E50），用于下跌/利空标注
- 中性色：浅灰（#95A5A6）
- 标题字色：近黑（#1A1A1A）

### 布局结构

页面为**单列、上下滑动**的网页报告，无封面页。内容从页面顶部直接开始，顶部是一条紧凑的「数据仪表盘横条」（dashboard strip），替代原框架的封面模块功能，承载关键数字和今日定性。往下依次是六个可折叠的分析模块。

```
[顶部仪表盘横条]   ← 不 sticky，随页面滚动消失
  指数收盘价+涨跌幅  |  成交额  |  情绪分  |  今日定性一句话

[Module 1] 宏观定价扫描
[Module 2] 市场情绪温度计
[Module 3] 板块结构性分析
[Module 4] 资金路线图
[Module 5] 技术形态与估值
[Module 6] 次日预判与历史镜像
```

布局基础契约（工程底线）：
- `body` 最大宽度 1180px，水平居中，内边距 32-40px
- `html` 与 `body` 背景色保持一致，避免居中后两侧色差
- 辅助文字不小于 11px，确保投屏可读
- 单文件输出，不引用本地资源，图表库通过 CDN 加载（ECharts）
- 顶部指数卡使用网格布局而非单行挤压；窄屏自动降为 2 列或单列
- 所有图表必须有固定高度容器；无数据时降级为说明卡片，不允许空白图表或元素重叠
- 最终 HTML 输出到项目根目录，JSON 中间产物继续保留在 `assets/`

### 交互与可视化要求

**顶部仪表盘横条**：
- 四个指数的数字卡片横向排列，涨跌用颜色区分，纯文字数字，无图表
- 情绪温度计用细长进度条展示（CSS progress bar），右侧标注分值和档位文字

**Module 2 情绪温度计**：
- 综合评分的5日趋势使用 ECharts 小型折线图（嵌入模块内）
- 散户/机构情绪对比使用两个并排的半圆形仪表盘（ECharts gauge）

**Module 3 板块分析**：
- 申万一级行业涨跌热力使用横向条形图（ECharts bar），按涨跌幅排序，颜色映射涨跌
- 轮动生命周期使用五阶段可视化进度条（纯 CSS），高亮当前阶段

**Module 4 资金路线图**：
- 资金迁移使用 ECharts 桑基图（Sankey）或简化流向箭头示意，展示板块间资金流动
- 超大单/小单对比使用分组柱状图

**Module 5 技术与估值**：
- 关键指数 MA 均线位置关系使用迷你 K 线图（ECharts candlestick，近30日）
- PE 历史百分位使用水平轨道滑块可视化（当前值在历史区间中的相对位置）
- ERP 股债性价比使用双轴折线图（股息率 vs 国债收益率，近2年历史）

**Module 6 次日预判**：
- 三情景概率使用水平分段条（三段不同颜色，类似投票结果），直观展示概率分布
- 每个情景卡片支持点击展开/收起触发条件详情（accordion 交互）

### 内容质量控制

渲染 Agent 在生成文本时必须遵守以下规范：

**禁用表达黑名单**（出现即重写）：
- 「市场情绪有所回暖」→ 必须给出情绪分数值及变化量
- 「需关注风险」→ 必须指出具体什么风险、什么信号触发
- 「可能震荡上行」→ 必须给出概率和区间
- 「技术面显示支撑」→ 必须指出具体支撑位及计算依据
- 「资金面较为活跃」→ 必须给出成交额和与历史均值的比较

**每份报告必须出现的量化锚点**：
- 情绪综合分（含散户/机构拆解）
- 资金迁移净额（流出 vs 流入板块）
- 当前 PE 历史百分位（沪深300）
- 关键支撑/压力位（含计算依据）
- 次日三情景概率（总和=100%，各附触发条件）
- 上期预判准确度评级（若有历史记录）

## 六、六个分析模块框架说明

> 以下为各模块的内容框架定义，供 Subagent 和渲染 Agent 参照执行。

### Module 1：宏观定价扫描

**核心问题**：今日行情背后的驱动力是什么？市场是否已充分定价？

包含三个子部分：

**有效事件过滤表**：从 tavily 拉取的新闻中，仅保留能够明确追溯到盘面印证的事件，按「事件→传导路径→受影响资产→定价完成度」四列呈现。定价完成度分三档：已充分 / 未充分（可能有滞后机会）/ 过度（存在回调风险）。

**预期差卡片（新增，建议固定展示 1-2 条）**：对当天最重要的主线，不只回答“发生了什么”，还要回答：
- 市场原预期是什么
- 今日新增信息是什么
- 哪些板块/资金实际响应了
- 当前属于未充分、局部充分还是过度交易
- 明日最重要的验证点是什么
- 哪个信号出现时，今天主叙事不成立

**宏观三维坐标系**：每日更新三个状态坐标，以简洁的状态标签+单行数据依据呈现——① 经济动能方向（改善/平稳/恶化）② 流动性环境（宽松/中性/收紧）③ 风险偏好（高/中/低）。坐标组合对应一种历史上的市场状态类比。

**隔夜外盘摘要**：美股三大指数、港股、关键大宗商品（原油、黄金、铜）的隔夜表现，以及对次日 A 股开盘的预期传导判断。

---

### Module 2：市场情绪温度计

**核心问题**：今日市场参与者的整体情绪处于什么水位？散户和机构是否分歧？

**综合情绪评分表**：7个维度的加权评分，合计100分。

| 指标 | 权重 | 散户/机构属性 |
|------|------|------------|
| 全市场成交额（与20日均值之比） | 20% | 综合 |
| 涨停/跌停比 | 15% | 散户偏向 |
| 涨停封板率 | 15% | 散户偏向 |
| 两融余额变化 | 15% | 散户偏向 |
| 北向资金净额 | 15% | 机构偏向 |
| 全市场换手率 | 10% | 综合 |
| 超大单净流入方向 | 10% | 机构偏向 |

基于各指标的散户/机构属性，分别汇总散户情绪代理分与机构情绪代理分，输出分歧或共识方向判断（共识多头 / 共识空头 / 散户热机构冷 / 散户冷机构热）。

**5日情绪趋势**：输出近5个交易日的情绪分序列和趋势方向，与近20日均值和近60日均值对比，判断当前是否处于「追涨风险区间」（高于60日均值一个标准差以上）。

---

### Module 3：板块结构性分析

**核心问题**：今日谁在涨、为什么涨、处于轮动的哪个阶段？

**行业强弱全景**：按申万一级行业分为强势/次强/中性/弱势四类，计算今日成长风格与价值风格的相对强弱（以成长代表性行业指数相对价值代表性行业指数的日收益差衡量）。

**TOP 2 板块深度解剖**：对当日涨幅和净流入均靠前的两个板块，从以下四个维度解剖：
- 驱动力来源分类（政策/业绩/主题/轮动承接/估值修复/事件催化）
- 轮动生命周期定位（启动/加速/高潮/分化/衰减五阶段，需给出判断依据）
- 资金主体识别（游资/机构/量化，依据各自的特征信号）
- 持续性评级（三星制，含评级理由）

**涨停板生态快照**：涨停总数、首板/连板分布、封板率、炸板数、主题集中度，最终给出当日「赚钱效应」的综合判断（强/中/弱），这是 A 股市场独有的情绪领先指标。

---

### Module 4：资金路线图

**核心问题**：钱从哪来、去了哪里、主力意图是什么？

**全市场资金结构**：按超大单/大单/中单/小单分类，输出各档净流入金额和占比，判断机构/散户行为组合类型（共识多头/共识空头/机构建仓散户减仓/机构减仓散户接盘）。

**资金迁移路线**：识别今日净流出额前三的行业和净流入额前三的行业，判断迁移性质（防御→进攻 / 进攻→防御 / 行业内轮动 / 整体出场），估算差额是否存在增量场外资金净进出。

**龙虎榜席位解读**：对触发龙虎榜条件的个股，识别买卖一席位的性质（知名游资/机构专用/量化营业部），给出主力意图判断。

---

### Module 5：技术形态与估值

**核心问题**：市场在哪里、贵不贵、下一步关键位在哪？

**技术状态矩阵**：沪深300/创业板指/科创50三个指数，各输出当前与MA5/20/60/250的距离（点数和百分比）、均线排列结构（多头/空头/粘合）、今日成交量与5日均量之比、今日K线形态名称。

**K线形态概率解读**：对识别出的K线形态，不做简单描述，而是给出基于历史统计的方向概率估算（如「长上影线在前期高位附近出现时，3个交易日内回调概率约65%」）。

**关键支撑/压力位**：对上证指数至少各给出两个支撑位和两个压力位，每个位置标注计算依据（MA类/斐波那契/成交密集区/历史高低点），并说明当前价格与各位置的距离。

**量价关系综合判断**：输出今日量价组合类型，区分日内涨幅与隔夜涨幅，判断资金行为性质（高开低走 = 情绪透支 / 低开高走 = 逢低承接）。

**估值水位表**：四个主要指数当前PE(TTM)及历史百分位，总体估值判断档位（历史低位P30以下/中位P30-P60/高位P60以上）。

**股债性价比（ERP）**：当前沪深300股息率 - 10年期国债收益率，与历史均值对比，判断当前股票相对债券的长期配置价值。

---

### Module 6：次日预判与历史镜像

**核心问题**：明天最可能发生什么、历史上类似时期之后怎样了？

**三情景预判**：

每个情景必须包含以下要素，缺一不可：
- 核心情景（概率最高，约55-65%）：最可能的市场状态，含预期指数区间和成交量区间
- 乐观情景（约20-30%）：需同时满足的具体触发条件（开盘30分钟内可观测），以及触发后的目标区间
- 悲观情景（约10-20%）：任意一条触发即成立的条件，以及触发后的风险应对参考
- 三情景概率之和必须等于100%

概率给出的依据必须可追溯（技术面依据+情绪依据+事件依据各至少一条），不接受模糊的主观判断。

新增要求：
- 每个情景都要写明“该情景在验证什么叙事”
- 每个情景都要写明“什么信号出现时，该情景失效”
- 次日复盘时优先用这些验证点和失效条件做检验，而不是事后补解释

**次日重点观测清单**：按优先级列出次日需关注的事项（重要数据发布时间、关键政策窗口、隔夜外盘方向等），让读者在开盘前有明确的观察议程。

**历史镜像**：基于当前估值百分位、情绪分、趋势状态三个维度，寻找历史上最相似的市场阶段，说明核心相似点和不可忽视的关键差异，引用历史参照期后续5日/10日/20日的平均表现（注明仅供参考，不构成预测）。

**上期预判回顾**（若有历史记录）：对上一期发布的三情景预判做命中率回顾，给出高/中/低准确度评级，若未命中则简要分析根因。积累近20期的命中率统计，形成自我校准机制（当发出「60%概率」情景时，实际命中率的校准修正值）。

## 七、语言规范

**先说好还是不好，再说为什么。** 每个模块的标题和首句都应是结论，而非描述。

保留英文的情况（仅限这些）：专业缩写（PE、PB、ERP、MA、ETF、MLF）、指数简称（A股、H股）、机构名称

必须中文的内容：所有分析结论、图表标题、评价性用语、方向性判断

禁止出现的表达：「总而言之」「需要指出的是」「值得注意的是」「总体来看」「综上所述」
